## Codebase Patterns
- Backend: FastAPI + SQLAlchemy async + Alembic
- AI Runtime: LangChain 1.x + LangGraph 1.x, executor.py에서 create_react_agent 사용 중
- 도구 타입: builtin, prebuilt, custom, mcp, skill_package
- MCP: httpx 직접 구현 (mcp_client.py), StructuredTool 수동 래핑 (tool_factory.py)
- 패키지 매니저: uv (backend), pnpm (frontend)
- 린트: ruff (line-length=100, target=py312)
- 테스트: pytest + aiosqlite in-memory
- DB: PostgreSQL 16 (docker-compose)

## S2 아키텍처 리서치 결과 (2026-04-05)

### SPEC 대비 실제 API 차이점
- `create_deep_agent(skills=[...])` → skills 파라미터 없음. backend=CompositeBackend(routes={"/skills/": FilesystemBackend(...)}) 사용
- `create_deep_agent(memory=[...])` → memory 파라미터 없음. backend CompositeBackend + store=PostgresStore(...) 조합
- `middleware` 파라미터는 확인됨: `middleware: Sequence[AgentMiddleware] = ()`
- 반환 타입 CompiledStateGraph — 기존 astream() 호환

### MCP 어댑터 패턴
- MultiServerMCPClient가 주 인터페이스: {"server_name": {"transport": "http", "url": "...", "headers": {...}}}
- get_tools() → list[BaseTool] 직접 반환 (수동 래핑 불필요)
- auth_config → HTTP headers 변환 필요 (api_key → Authorization, jwt_token → Bearer)

### ADR-001 핵심 설계
- build_agent() → create_deep_agent 단일 호출 (폴백 제거)
- MCP 도구: 서버 URL별 그룹화 → MultiServerMCPClient → get_tools() → 이름 필터링
- builtin/prebuilt/custom 도구 생성은 기존 tool_factory.py 유지
- streaming.py는 변경 없음 (astream API 동일)

## M3-S2 아키텍처 설계 결과 (2026-04-06) — Pichai

### Backend 선택: FilesystemBackend
- root_dir="./data", virtual_mode=True
- CompositeBackend 불필요 (스크래치패드 미사용)
- StoreBackend 불필요 (PoC에서 cross-thread 메모리 불필요)
- 심볼릭 링크 불가: FilesystemBackend는 O_NOFOLLOW로 symlink 차단

### Skills 경로 매핑
- skills=["/skills/"] → _list_skills가 data/skills/ 하위 서브디렉토리 자동 탐색
- _list_skills 동작: ls_info(source_path) → is_dir 필터 → {dir}/SKILL.md 다운로드 → frontmatter 파싱
- **주의**: source_path는 **스킬 컬렉션 디렉토리** (부모). 개별 스킬 디렉토리 직접 전달 불가
- Per-agent 필터링 없음 (PoC). 모든 에이전트가 /skills/ 전체 탐색. 프로그레시브 디스클로저로 보완.

### Memory 경로 패턴
- memory=["/agents/{agent_id}/AGENTS.md"]
- MemoryMiddleware: download_files() → file_not_found 시 무시 (에러 없음)
- data/agents/{agent_id}/ 디렉토리는 에이전트 생성 시 생성

### Text 스킬 처리
- text 스킬은 DB content 필드에만 존재 → 디스크에 SKILL.md로 물질화 필요
- skill_service에서 생성/수정 시 _materialize_skill_to_disk() 호출
- DB content = source of truth, SKILL.md = 파생물

### build_agent() 변경

- create_deep_agent에 그대로 전달
