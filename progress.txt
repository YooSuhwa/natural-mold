## ADR-008 핵심 결정 (반드시 지킬 것)

- **user_id**: NOT NULL FK `users.id` (권한 분리의 기반)
- **UNIQUE 제약 없음**: UUID PK + user_id FK + 인덱스 `(user_id, type, provider_name)`만
- **type**: VARCHAR(20) `'prebuilt' | 'mcp' | 'custom'`
- **provider_name**: VARCHAR(50). Pydantic validator:
  - `type='prebuilt'`: credential_registry enum (naver, google_search, google_workspace, google_chat, custom_api_key) 5종만
  - `type='mcp'` / `type='custom'`: 자유 문자열 (영문/숫자/언더스코어, `^[a-z0-9_]+$` 권장)
- **display_name**: VARCHAR(200) — UI 레이블 (중복 허용, UNIQUE 없음)
- **credential_id**: UUID nullable FK `credentials.id` ON DELETE SET NULL
- **extra_config**: JSON nullable. MCP만 사용:
  - `{url, auth_type, headers?, env_vars?, transport?, timeout?}`
  - `url`, `auth_type`은 필수 (MCP인 경우)
  - PREBUILT/CUSTOM은 NULL
- **is_default**: BOOLEAN NOT NULL DEFAULT false. 같은 (user_id, type, provider_name) 범위에서 1개만 True. 서비스 레벨에서 원자적 토글 (기존 default 자동 해제)
- **status**: VARCHAR(20) NOT NULL DEFAULT 'active' — `'active' | 'disabled'`
- **created_at, updated_at**: TIMESTAMP NOT NULL
- **인덱스**: `(user_id, type, provider_name)` (hot path 조회)

## API 계약 (젠슨/베조스용)

```
GET    /api/connections                     # 현재 유저의 connection 목록 (필터: type, provider_name)
GET    /api/connections/{id}                # 단일 조회 (user_id 일치 안 하면 404)
POST   /api/connections                     # 생성 (body: ConnectionCreate)
PATCH  /api/connections/{id}                # 수정 (body: ConnectionUpdate)
DELETE /api/connections/{id}                # 삭제
```

모든 엔드포인트: `get_current_user` 필수 + user_id 필터.
