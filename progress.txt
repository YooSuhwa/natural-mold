- `services/spend_writer.py` — `SpendEntry` dataclass + `DailySpendUpdateQueue` (asyncio.Queue + 백그라운드 drain task). 핵심 결정:
  - 단일 큐로 3개 axis 모두 처리 (분리하면 producer가 3번 enqueue 해야 함)
  - flush 트리거: `flush_interval_sec=5.0` 또는 `batch_size=100` 도달 시
  - drain: `wait_for(queue.get(), timeout=interval)` → greedy `get_nowait` until batch_size → `_flush_batch`
  - `_flush_batch`: entries를 axis별 `(date, target_id)`로 group → 각 테이블에 dialect-aware `INSERT ... ON CONFLICT (date, target_id) DO UPDATE SET total_X = X + EXCLUDED.X` (PG/SQLite). 그 외 dialect는 application-level upsert fallback.
  - producer (`add()`) 비차단: `put_nowait` (QueueFull / RuntimeError 시 warning + drop, agent 실행 막지 않음).
  - `stop()` graceful: 큐 잔여 flush → task cancel → await (with `contextlib.suppress`).
  - 모듈 전역 `spend_queue` 싱글턴.
- `hooks/builtin/spend_hook.py` — `agent_invoke` post hook에서 `spend_queue.add(SpendEntry(...))`. 0 token / 0 cost row는 skip (interrupt 등 비정상 종료 가드). failure isolation은 HookRegistry + 자체 try/except 이중 가드.
