## Codebase Patterns (M1~M4에서 확인된 것)
- 모델 컬럼 추가 시 Pydantic 응답/요청 + FE 타입 세트 체크 (M3 provider_name, M4 connection_id 두 번 다 블로커)
- Alembic revision ID 32자 이하 (alembic_version VARCHAR(32))
- `is_default` partial unique index — user×type×provider당 1개
- PG 네이티브 SQL 이관은 aiosqlite 테스트에서 `inspect.getsource` 계약 가드로만 검증
- React Query setQueryData seed 패턴 (M4 useCreateConnection.onSuccess) — invalidate 대신 seed로 race 방지
- find-or-create 패턴: `useConnections({type, provider_name})` + filter → 없으면 CredentialFormDialog → useCreateConnection

## M5 Spec v2 업데이트 (2026-04-19, 베조스 S1 5개 항목 반영)

- **확장 관점**: ConnectionBindingDialog는 신규 아닌 M3 기존 셸 **확장**. `type` discriminated union + 신규 props는 기존 `onSaved` alias 유지로 PREBUILT 호출처 후방 호환
- **Props 결과 shape (discriminated)**:
  - PREBUILT/CUSTOM: `onBound({ kind: 'connection', connection })`
  - MCP: `onBound({ kind: 'mcp-credential', serverId, credentialId })` — Connection 엔티티 미생성
  - 기존 `onSaved(connection)`는 legacy alias로 유지
- **CUSTOM bridge 보존 정책 (M5)**: 기본 저장은 `connection.credential_id`만 PATCH, `tool.credential_id`는 건드리지 않음. UI는 bridge 상태 경고 배너 + "공통 connection으로 복귀" 옵션 버튼만 제공. override clear는 M6에서 일괄
- **MCP 스코프 축소 (베조스 §2.2 권장 채택)**: `type='mcp'` 셸은 **credential binding만** 담당. server create/edit는 기존 add-tool-dialog / mcp-server-rename-dialog 유지. Section A(URL/transport) 와이어프레임 제거, CredentialSelect-only로 단순화. props에 `mcpServerId` 추가. 내부 저장은 `useUpdateMCPServer({credential_id})`
- **/connections MCP 섹션 "연결 추가" CTA**: ConnectionBindingDialog 진입 X. 대신 `add-tool-dialog` MCP 탭을 연다. `AddToolDialog`에 `defaultTab='mcp'` prop 추가는 허용 (드라이브바이 최소 한 줄)
- **Credential 삭제 semantics (N:1 반영)**:
  - Connection 삭제 ≠ Credential 삭제. connection row만 제거, credential row는 살아남음 (고아 허용)
  - Credential 삭제는 `CredentialFormDialog` 내 삭제 버튼에서만 — 다른 connection 참조 중이면 disabled
  - Drawer "삭제" 라벨은 "이 연결만 삭제 (credential은 유지)"로 명시
  - drawer의 credential 관련 action은 [변경]/[편집]/(⋮→[삭제]) 3종 — 파괴 동작은 한 단계 숨김
- **tools/page.tsx getAuthStatus 회귀 주의 (§12.1 + §11b.1)**: CustomAuthDialog 교체 시 `tool.credential_id` 단독 판정이면 bridge 보존 원칙상 회귀 없음. 신규 `tool.credential_id IS NULL AND connection_id NOT NULL` row 등장 시 판정 로직을 `connection_id`로 확장 필요. S5 수동 E2E 3종 (bridge/신규/legacy-only) 필수
- **useToolsByConnection 파생 selector**: CUSTOM은 `tool.connection_id` 매칭, MCP는 `tool.mcp_server_id → mcp_server.credential_id → connection.credential_id` 이중 hop. type별 분기 필수
- **PREBUILT 호출처 후방 호환**: `type` narrowing + `isPrebuiltProviderName` guard 유지, `useConnections`는 type='mcp' 시 스킵
