## Codebase Patterns
- Next.js 16 + React 19 + Tailwind CSS v4 + shadcn/ui (Base UI)
- 상태: TanStack Query (서버) + Jotai (클라이언트)
- i18n: next-intl (useTranslations, useFormatter)
- 아이콘: lucide-react
- 토스트: sonner
- 테마: next-themes
- 프론트엔드 경로: /Users/chester/dev/natural-mold/frontend/
- 패키지 매니저: pnpm
- 빌드: pnpm build, 린트: pnpm lint

## Gotchas
- Next.js 16은 기존과 API가 다를 수 있음 — frontend/node_modules/next/dist/docs/ 확인 필요
- params, searchParams가 Promise로 변경됨 (use() 사용)
- shadcn/ui는 @base-ui/react 기반 (headless)
- Separator 컴포넌트: @/components/ui/separator
- Slider, Checkbox 컴포넌트가 shadcn에 있는지 확인 필요 (없으면 설치)
- 도구 페이지: 카드 onClick과 내부 버튼 onClick이 충돌 — stopPropagation 필수
- i18n 메시지 파일 위치: frontend/messages/ 아래 ko.json, en.json 등

## 실패 교훈
- 브레드크럼에서 useTranslations() fallback 시 t(segment)를 직접 호출하면 MISSING_MESSAGE 에러 발생 → ROUTE_LABELS에 없으면 raw segment를 그대로 표시해야 함
- CardFooter에 onClick stopPropagation 추가하면 카드 내부 모든 버튼 이벤트 버블링을 한 번에 차단 가능

## UX 디자인 스펙 (tim-cook, 2026-04-07)
- 스펙 문서: docs/design-docs/ux-improvement-spec.md
- 6개 항목: Coming Soon 패턴, 에이전트 카드 리디자인, 설정 탭 분리, 브레드크럼, 앱 설정 페이지, 도구 상세 Dialog
- 오퍼시티 전략 2가지: Dim(opacity-50/70 = 미출시), Show/Hide(opacity-0/100 = 노이즈 줄임)
- 트랜지션 표준: duration-200, ease-out
- shadcn/ui 컴포넌트 네이밍 검증 완료 (TabsTrigger ✓, Dialog* ✓)
- 설정 페이지 4탭 분리: basic/model/tools/triggers + sticky 저장바
- 도구 상세: Sheet → Dialog 변경 (패턴 통일)
