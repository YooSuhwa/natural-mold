## Codebase Patterns
- 프론트엔드: Next.js 16, React 19, Tailwind CSS v4, shadcn/ui, lucide-react
- 상태관리: Jotai (채팅 스트리밍), TanStack Query (서버 데이터)
- 채팅 SSE: 커스텀 프로토콜 (content_delta, tool_call_start, tool_call_result, message_end)
- 채팅 컴포넌트 위치: frontend/src/components/chat/
  - chat-input.tsx (82줄) — textarea + 전송 버튼
  - message-bubble.tsx (106줄) — user/assistant/tool 메시지 렌더링
  - streaming-message.tsx (72줄) — 실시간 스트리밍 표시
  - tool-call-display.tsx (90줄) — 도구 호출 상태/결과 표시
  - markdown-content.tsx (84줄) — react-markdown 렌더링
  - conversation-list.tsx (251줄) — 대화 목록 사이드바
- 채팅 페이지: frontend/src/app/agents/[agentId]/conversations/[conversationId]/page.tsx (200줄)
- 채팅 스토어: frontend/src/lib/stores/chat-store.ts (streamingMessageAtom, streamingToolCallsAtom, isStreamingAtom)
- 백엔드 SSE: backend/app/agent_runtime/streaming.py
- UI 프레임워크: shadcn/ui + lucide-react 아이콘
- 폰트: Geist / Geist Mono
- 패키지 매니저: pnpm (frontend), uv (backend)

## 발견된 문제점 (agent-browser 캡처 2026-04-06)
- P0: 도구 결과가 raw Python dict로 표시됨 {'type': 'text', 'text': '...'} → text 필드만 추출해야 함
- P0: \xa0\n 제어 문자가 도구 결과에 노출
- P0: 이미지 크기 제한 없음 (전체 너비 차지)
- P1: 토큰/비용 표시 없음 (MessageBubble에 tokenInfo UI 준비됨, 데이터 미연결)
- P1: 입력 영역 빈약 (textarea + 전송 버튼만)
- P1: 코드 블록 문법 강조 없음
- P2: 어시스턴트 메시지에 봇 아이콘 없음
- P2: 메시지 간 간격 불일치

## Gotchas
- Next.js 16 사용 — frontend/node_modules/next/dist/docs/ 참조
- Tailwind CSS v4 — @import 방식, CSS 변수 기반 테마
- 채팅 SSE는 AsyncGenerator 패턴 (frontend/src/lib/sse/stream-chat.ts)
- message_end 이벤트에 usage 필드 있음 (input_tokens, output_tokens 등)
- SSEEvent 타입: frontend/src/lib/types/index.ts (260-274줄)
- MessageBubble의 tokenInfo prop: { tokens: number; cost: number } | null
- shadcn/ui Dialog 사용 가능 (이미지 확대용)
- 마크다운 img 태그에 이미 lazy loading + URL 변환 로직 있음
