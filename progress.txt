## 프로젝트 목적

M1에서 `connections` 테이블을 parallel run으로 만들었고, 이제 M2에서 MCP 도구의 credential/서버 설정 해석 경로를 connection 경유로 전환한다. 기존 `mcp_servers` 테이블은 read-only deprecate (M6에서 drop).

**이 M2가 끝나면**: MCP 도구 실행 시 `tool.connection_id → connections → credential/extra_config` 경로로 동작. `mcp_servers` 직접 참조하는 경로는 legacy fallback으로만 남음.

## ADR-008 반영 규칙 (반드시 지킬 것)

- **extra_config 구조**: `{url, auth_type, headers?, env_vars?, transport?, timeout?}`
- **env_vars 값**: 반드시 `${credential.<field_name>}` 템플릿 (평문 금지). 런타임 해석 시 `credential.data_decrypted[<field_name>]`으로 치환
- **connection.credential_id ON DELETE SET NULL**: credential 삭제 시 connection 살아있음
- **tool.connection_id**: nullable FK. M6까지 `tool.mcp_server_id`와 공존
- **legacy fallback**: M6 전까지 `connection_id IS NULL AND mcp_server_id IS NOT NULL`인 tool은 기존 경로로 동작 (이관 실패 tolerance)
