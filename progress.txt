## Codebase Patterns

### Backend (FastAPI + SQLAlchemy async)
- Router → Service → Model 계층, 비즈니스 로직은 services/
- 모든 DB 접근은 async (`AsyncSession`)
- 의존성 주입: `Depends(get_db)`, `Depends(get_current_user)` (mock user, app/dependencies.py)
- 스키마: Pydantic, `model_config = {"from_attributes": True}`
- PATCH 부분 업데이트: `data.model_dump(exclude_unset=True)`로 null vs missing 구분
- 에러 코드: `app/error_codes.py`에 정의된 헬퍼 사용 (mcp_server_not_found 이미 존재)
- ToolType StrEnum: BUILTIN/PREBUILT/CUSTOM/MCP

### Frontend (Next.js 16 + React 19 + TanStack Query + Jotai)
- ⚠️ Next.js 16 — 기존 Next.js와 다름. 코드 작성 전 frontend/AGENTS.md 확인
- API 클라이언트: lib/api/*.ts (apiFetch 래퍼)
- TanStack Query hooks: lib/hooks/use-*.ts
- queryKey invalidate 시 ['tools']와 ['mcp-servers'] 둘 다 무효화 (도구 카드 인증 표시 갱신)
- shadcn/ui: components/ui/ — Accordion/Collapsible 미설치 (자체 구현 사용)
- 'use client' 최소화, any 금지 → unknown + 타입 가드
- i18n: frontend/messages/ko.json + next-intl, useTranslations('tool.page') 패턴

## 핵심 데이터 모델 (사전 확인)

- `MCPServer.tools`: `cascade="all, delete-orphan"` → 서버 삭제 시 도구 자동 cascade (`backend/app/models/tool.py:52-54`)
- `MCPServer.credential_id`: `ondelete="SET NULL"` (`tool.py:43-45`)
- `tool_service.get_mcp_servers()` 이미 정의됨 (`tool_service.py:119-126`), 라우터 미노출
- `chat_service.build_tools_config()` precedence (`chat_service.py:171-190`): 현재 tool.cred → server.cred → tool.auth → server.auth
- `credential_service.resolve_server_auth()` 헬퍼 존재 (`credential_service.py:106-115`) → 재사용

## Gotchas

- 기존 `POST /api/tools/mcp-server` (단수)는 그대로. 새 라우트만 복수형. 일관성 어색하지만 호환성 우선
- `MCPServerListItem.credential` 응답에 CredentialBrief (id/name/provider_name) 포함 — data_encrypted 노출 금지
- `Tool.mcp_server_id` FK는 DDL ondelete 없음 → ORM cascade에 의존. raw SQL DELETE는 안 됨
- mcp-auth-dialog.tsx 삭제 시 import 위치 확인 필수 (현재는 tools/page.tsx만 import)
- 사용자 메모: 별도 브랜치 작업 시 stash 대신 git worktree 사용 (이번엔 main에서 modified만 있어서 일반 브랜치)
- ENCRYPTION_KEY 미설정 시 credential 생성 거부 (503)

## 실패 교훈

- 백엔드 M1 완료 시점에 추가:
  - `list_mcp_server_items`는 `Tool` count 서브쿼리 + LEFT OUTER JOIN으로 단일 쿼리. `MCPServer.credential` lazy="joined"라 추가 selectinload 불필요.
  - `chat_service.build_tools_config` MCP precedence 분리 시, 비MCP 분기에서 `tool.mcp_server.auth_config` 폴백을 함께 제거해야 함 (이전엔 `tool.type == MCP and ...` 가드로 묶여 있었음).
  - 새 테스트에서 `Credential.data_encrypted`는 평문 JSON으로 두고 `chat_service.resolve_credential_data`를 patch.object로 모킹하면 ENCRYPTION_KEY 유무와 무관하게 결정적임.
