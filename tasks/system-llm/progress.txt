## API 계약 (사티아 확정 — 젠슨/저커버그 합의 기준)

role enum: `text_primary` | `text_fallback` | `image`

GET  /api/system-llm-settings        (super_user)
  → 200: [ SystemLlmSettingOut x3 ]
PUT  /api/system-llm-settings/{role} (super_user)
  body: { "credential_id": uuid|null, "model_name": str|null }
  → 200: SystemLlmSettingOut
  - credential_id 지정 시: is_system=True && definition_key ∈ {openai,anthropic,openrouter,openai_compatible} 검증.
    위반 시 404(없음)/422(타입불가) — enumeration oracle 방지 위해 메시지 통일.

SystemLlmSettingOut:
  { role: str, credential_id: uuid|null, credential_name: str|null,
    provider: str|null (=credential.definition_key), base_url: str|null,
    model_name: str|null, configured: bool, updated_at: datetime }

discover (기존 재사용, 변경 금지):
  POST /api/credentials/{credential_id}/discover-models → 모델 목록

## Codebase Patterns (확인된 사실)
- Router→Service→Model, async SQLAlchemy 2.0 select(). DI: Depends(get_db, require_super_user, verify_csrf).
- 권한: super_user 전용은 require_super_user. is_system 리소스는 user_id NULL.
- credential 복호화: app.credentials.service.decrypt_with_external(data_encrypted) → payload dict.
  - LLM credential payload key: "api_key" 또는 "token". openai_compatible/openrouter는 "base_url" 보유.
- find_system_by_definition(db, definition_key) 존재. get_system 류 헬퍼 확인 후 사용/추가.
- model_factory.create_chat_model(provider, model_name, api_key=None, base_url=None, **extra).
  - _OPENAI_FAMILY_BASE_URLS: openai/openrouter canonical pin. openai_compatible은 base_url 필수.
- LLM credential definition_key: anthropic, openai, openrouter, openai_compatible (definitions/__init__.py).
- 마이그레이션 head: m44 (ADR-018). 신규는 m45, down_revision='m44_relative_storage_path' 확인.
- 테스트: pytest + aiosqlite in-memory. 새 테이블은 conftest 모델 import 경로 확인.

## S2 테스트 사전리뷰 (베조스, 2026-05-26 — test_system_llm_settings.py 11 PASS 기준)
커버 양호: GET 3-role, PUT happy(openai/openai_compatible base_url passthrough/anthropic), PUT clear, 비존재 cred 404, 비-LLM 422, user-cred 404 거부, resolver configured/unconfigured/model_name누락.
갭(→젠슨 '?' 전달):
- HIGH require_super_user negative 테스트 부재: conftest client fixture가 항상 is_super_user=True+CSRF bypass(L108,132). 일반유저 GET/PUT 거부(403/404) 검증 불가 → non-super client fixture 추가 또는 raw_client+실제 JWT 필요. ADR결정4 핵심 가드인데 무검증.
- HIGH credential ON DELETE SET NULL 동작 테스트 부재 + aiosqlite는 PRAGMA foreign_keys=ON 미설정(conftest engine L44, connect listener 없음) → 테스트 추가해도 SET NULL 발화 안 함. event.listens_for(engine.sync_engine,"connect")로 PRAGMA 켜야 의미있는 회귀가드.
- MEDIUM role UNIQUE 제약 직접 테스트 부재(중복 role insert→IntegrityError 기대). SQLite UNIQUE는 FK pragma 없이도 enforce되니 추가 가능.
- MEDIUM CHECK(role IN) DB레벨 테스트 부재 — 현재 test_put_unknown_role_404는 router검증만. role="bogus" 직접 insert→IntegrityError로 defense-in-depth 확인 권장.
- MEDIUM 404/422 detail 동일성 assert 약함: 양쪽 substring("system LLM credential")만 체크, byte-identical 미검증 → 422 메시지 변경돼도 통과. 두 응답 detail 캡처 후 == assert로 enumeration oracle 회귀가드 강화.
- LOW updated_at PUT 갱신 테스트 부재.
