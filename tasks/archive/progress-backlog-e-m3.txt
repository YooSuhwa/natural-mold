## 프로젝트 목적

M1에서 `connections` 테이블 + CRUD API, M2에서 MCP 도구를 connection 경유로 이관했다. M3에서는 PREBUILT 도구가 per-user 기준으로 각자의 credential을 쓰도록 한다. 현재는 `tools.credential_id` (공유 행) 참조 → 유저 A의 credential이 유저 B에게 노출되는 구조.

**이 M3가 끝나면**: PREBUILT 도구 실행 시 `tool.provider_name + current_user → default connection → credential` 경로로 동작. 같은 seed tool을 여러 유저가 각자의 credential로 실행 가능.

## ADR-008 / M0 합의 반영 규칙

- **Tool.provider_name 신설**: PREBUILT는 필수(seed에서 부여), MCP/CUSTOM은 NULL
- **Connection default 해석**: user_id + type='prebuilt' + provider_name + is_default=true (per-user, per-provider 1개)
- **env fallback 유지**: default connection 없으면 `cred_auth = {}` → 기존 `settings.naver_*` 등 경로 유지 (ADR-008 §11)
- **mock user 시드**: Alembic m10에서 env 값 → credential → default connection 자동 생성 (idempotent)
- **Legacy fallback**: `tools.credential_id`는 PREBUILT에서 무시하는 게 원칙이나, M6까지는 `provider_name IS NULL` 케이스에 한해 기존 경로 유지 (이행 tolerance)
- **Cross-tenant 가드**: `assert_connection_ownership` / `assert_credential_ownership`는 M2에서 이미 도입 — PREBUILT에서도 재사용

## Tool-Provider 매핑 (m10 백필용)

| Tool name 패턴 | provider_name |
|----------------|---------------|
| `Naver *` | naver |
| `Google Search`, `Google Image Search`, `Google News Search` | google_search |
| `Gmail *`, `Google Calendar *` | google_workspace |
| `Google Chat *` | google_chat |
| 그 외 `type='prebuilt'` | WARN + NULL 유지 (수동 수정 필요) |
| `type='builtin'` (API 키 불필요) | NULL (connection 없음) |

## Alembic 주의사항

- Alembic head 확인: `cd backend && uv run alembic heads` → `m9_migrate_mcp_to_connections`
- 신규 `m10_prebuilt_connection_migration.py` `down_revision = "m9_migrate_mcp_to_connections"`
- **idempotent seed**: 이미 `(user_id, type='prebuilt', provider_name, is_default=true)` connection이 있으면 skip (partial unique index가 걸려 있어 insert 실패를 피해야 함)
- downgrade는 마이그레이션이 생성한 연결만 삭제 (사용자 수동 생성분은 건드리지 않음 — `display_name` 또는 별도 마커로 구분)
- SQLite drop_column은 `op.batch_alter_table` 래핑 필수
