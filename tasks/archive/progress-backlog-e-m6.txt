# progress.txt — 백로그 E M6 (Cleanup — backend drop + legacy 제거)

## Codebase Patterns (M1~M5에서 확인된 것)

- Alembic convention: `m{N}_<snake_case>.py`. 최신 head는 `m11_custom_credential_migration`.
- FK constraint naming: `fk_<table>_<col>` 패턴 사용. drop 시 명시적 constraint name.
- aiosqlite in-memory 테스트 — 실제 FK cascade 동작이 PG와 다를 수 있음. 통합 검증은 docker PG 사용.
- Connection 엔티티는 1:1(CUSTOM) 또는 N:1(MCP 공유) credential 참조. M5 기준 is_default=True인 PREBUILT connection이 tool.connection_id와 매핑.
- `status='active' | 'disabled'` kill-switch. disabled → fail-closed at chat_service.

## Gotchas

- `tools.credential_id` FK ondelete=SET NULL → drop 시 FK constraint 먼저 제거 필수
- `tools.mcp_server_id` FK는 ondelete RESTRICT일 수 있음 — 드롭 전 확인
- `chat_service.py` `merged_auth = {**cred_auth, **(link.config or {})}` 라인 — `link.config`는 `agent_tools.config`. 실제로 저장된 값이 있는지 S1에서 재확인
- `ToolResponse._mask_auth_config`는 auth_config 컬럼 제거와 함께 삭제. 다른 곳에서 참조 없는지 grep
- `lib/api/tools.ts` `updateAuthConfig` 함수 — 호출처가 모두 dead인지 M5 이후 상태 확인 (UI 통합 후 3개 legacy dialog 삭제됨 → 호출처 0일 가능성 높음)
- `use-chat-runtime.ts:74` streamError unused warning은 M5 알려진 부채. M6에서 건드리지 말 것
- `credentials.ts` / `use-credentials.ts`는 UI 분리되었지만 Connection 내부 API로 살아있음. 지우지 말 것
