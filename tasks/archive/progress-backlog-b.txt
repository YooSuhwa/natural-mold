## Codebase Patterns (이전 PR에서 확립)

### Backend (FastAPI + SQLAlchemy async)
- Router → Service → Model 계층
- 의존성 주입: `Depends(get_db)`, `Depends(get_current_user)` (mock user)
- PATCH 부분 업데이트: `data.model_dump(exclude_unset=True)` + `_apply_credential_update` 헬퍼 (이미 존재)
- ToolType StrEnum: BUILTIN/PREBUILT/CUSTOM/MCP
- IDOR 방어: user_id 미스매치 시 None 반환 → 라우터에서 404
- ToolResponse는 `auth_config`를 마스킹 (`***`) — round-trip 거부 validator 적용됨

### Frontend (Next.js 16 + React 19 + TanStack Query)
- shadcn/ui: components/ui/
- API 클라이언트: lib/api/*.ts (`apiFetch` 래퍼)
- TanStack Query hooks: lib/hooks/use-*.ts
- queryKey invalidate: PATCH는 ['tools']만, register/delete만 ['mcp-servers']까지
- 'use client' 최소화, any 금지 → unknown + 타입 가드
- i18n: frontend/messages/ko.json + next-intl

## Gotchas

- 이전 PR과 동일하게 ENCRYPTION_KEY 필수 (이미 설정됨)
- 마스킹 sentinel `***`는 PATCH로 다시 보내면 422 거부됨 — UI는 sentinel 값 다시 제출 안 함
- 사용자 메모: 별도 브랜치 작업 시 stash 대신 worktree 사용 (이번엔 main에 modified 없으므로 일반 브랜치)

## 실패 교훈

- M1 변경(`update_tool_auth_config`에 CUSTOM 허용) 시 `tests/test_tools_router_extended.py::test_update_auth_config_non_prebuilt_returns_404`가 회귀로 잡힘. 이 테스트의 전제(=CUSTOM 거부)가 의도적으로 뒤집힌 케이스이므로 "회귀 아님" — 새 동작(다른 사용자 CUSTOM → 404 IDOR)을 검증하도록 갱신함. 추후 비슷한 의미 반전 변경 시 매칭되는 기존 테스트 1차 grep 권장 (예: `non_prebuilt`, `not in.*PREBUILT` 등 키워드).
- 신규 테스트는 `test_update_mcp_server_credential` 패턴을 그대로 차용 (monkeypatch encryption._get_fernet → None, Credential 시드, PATCH 검증, 다른 user_id 거부). `_seed_mcp_server_with_tools` 같은 헬퍼는 CUSTOM에 없으므로 인라인 시드.
- (M2) `frontend/tests/components/tool/add-tool-dialog.test.tsx`에 `Bearer` radio + `sk-xxx` placeholder를 검증하는 테스트 2건이 inline auth 제거로 깨졌다. 신규 credential UI(인증 없음 디폴트 + payload에 credential_id 미포함)에 맞게 갱신해서 11/11 PASS 회복. M1과 동일 패턴 — 의미 반전 시 keyword grep으로 잠재 깨짐 테스트를 먼저 찾는 것이 효율적.
- AddToolDialog는 `CredentialFormDialog`를 MCP/custom 두 탭이 공유한다. `onCreated` 콜백이 단일이라 어느 탭이 열었는지 추적이 필요 — `credentialTarget: 'mcp' | 'custom'` state로 분기 처리. 단일 다이얼로그를 두 트리거가 공유할 때 일반화 가능한 패턴.
