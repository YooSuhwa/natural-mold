## 📋 시스템 프롬프트 작성 가이드

시스템 프롬프트를 작성할 때 다음 구조를 따르면 좋습니다:

```markdown
# {에이전트 이름}

## Role (역할)
[1-2 문장으로 에이전트의 목적 설명]

## Responsibilities (주요 책임)
[번호 매긴 작업 목록]

## Tool Guidelines (도구 사용 가이드)
### `{도구명}`
- Purpose: [기능 설명]
- When: [언제 사용할지]
- Caution: [주의사항]

## Subagent Guidelines (서브에이전트 가이드)
### `{서브에이전트명}`
- Expertise: [전문 분야]
- Delegate when: [언제 위임할지]

## Workflow (작업 흐름)
[단계별 프로세스 설명]

## Constraints (제약사항)
- ALWAYS: [반드시 해야 할 것]
- NEVER: [절대 하면 안 될 것]
```

## 💡 작성 팁

### 1. **Role (역할)** 작성
- 에이전트가 **무엇을 하는지** 명확히
- 1-2 문장으로 간결하게
- 예: "사용자의 질문에 대해 인터넷 검색을 통해 최신 정보를 수집하고, 신뢰할 수 있는 답변을 제공하는 에이전트입니다."

### 2. **Responsibilities (책임)** 작성
- 에이전트가 **수행해야 할 구체적인 작업** 나열
- 번호를 매겨서 순서 명시
- 예:
  ```
  1. 사용자의 질문을 분석하여 검색 키워드 결정
  2. tavily_search 도구를 활용하여 인터넷에서 정보 검색
  3. 검색 결과를 종합하여 명확한 답변 제공
  ```

### 3. **Tool Guidelines (도구 가이드)** 작성
- 각 도구마다 **3가지 정보** 포함:
  - **Purpose**: 도구의 기능
  - **When**: 언제 사용할지 (트리거 조건)
  - **Caution**: 주의할 점
- 예:
  ```
  ### `tavily_search`
  - Purpose: 인터넷에서 최신 정보 검색
  - When: 사용자가 정보를 요청할 때, 최신 뉴스가 필요할 때
  - Caution: 검색 쿼리는 명확하게, 신뢰할 수 있는 출처 우선
  ```

### 4. **Workflow (작업 흐름)** 작성
- **단계별 프로세스** 설명
- 도구 사용 순서 명시
- 예:
  ```
  1. 질문 분석
  2. 검색 실행
  3. 결과 검토
  4. 답변 작성
  ```

### 5. **Constraints (제약사항)** 작성
- **ALWAYS**: 반드시 지켜야 할 행동
- **NEVER**: 절대 하면 안 될 행동
- 예:
  ```
  - ALWAYS: 검색 결과에 기반한 정보만 제공
  - NEVER: 신뢰할 수 없는 출처 사용
  ```

## 🎯 프롬프트 작성 시 고려사항

| 항목          | 설명                                     |
| ------------- | ---------------------------------------- |
| **명확성**    | 모호한 표현 피하기, 구체적으로 작성      |
| **구조**      | 섹션을 명확히 구분, 계층 구조 사용       |
| **도구 연계** | 에이전트가 사용할 도구를 명시적으로 언급 |
| **제약 명시** | 해야 할 것과 하면 안 될 것을 명확히      |
| **예시 포함** | 필요시 구체적인 예시 추가                |

## 📌 실제 예시

현재 만든 "인터넷 검색 에이전트"의 프롬프트가 이 구조를 따르고 있습니다:
- ✅ Role: 인터넷 검색을 통해 정보 제공
- ✅ Responsibilities: 4가지 책임 명시
- ✅ Tool Guidelines: tavily_search 사용법 상세 설명
- ✅ Workflow: 5단계 프로세스
- ✅ Constraints: ALWAYS/NEVER 규칙

이 템플릿을 기반으로 다양한 에이전트를 만들 수 있습니다! 🚀
